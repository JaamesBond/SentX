## How to Contribute Now
- Review architecture documents for feedback
- Suggest detection scenarios
- Share relevant OT security resources
- Report gaps in threat model

## Future Contribution Areas
- Detection rule development (Sigma, Suricata, Zeek scripts)
- Threat intelligence feed integration
- Dashboard components
- Documentation improvements

## Standards
- All detection logic must map to MITRE ATT&CK for ICS
- OT safety considerations are non-negotiable
- Code must include inline comments explaining "why", not just "what"
